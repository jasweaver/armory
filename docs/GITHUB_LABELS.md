# GitHub Labels

## Priority

- `P0: Critical`
- `P1: High`
- `P2: Medium`
- `P3: Low`

## Type

- `bug`
- `feature`
- `enhancement`
- `refactoring`
- `testing`
- `documentation`

## Workflow

- `in-progress`
- `needs-review`
- `blocked`

## Usage Rules

- Every issue should include one priority label
- Every issue should include at least one type label
- Add workflow labels as status changes
- Keep labels minimal and non-duplicative
